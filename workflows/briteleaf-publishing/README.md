# Briteleaf Publishing Pipeline

This workflow defines how one Briteleaf topic moves from idea to published article to supporting distribution assets.

## Goal

Turn one strong topic into a full publishing stack that supports traffic, clarity, and repurposing.

## Core Principle

One topic should produce:
- one primary article
- multiple short-form assets
- platform-specific support content
- a repeatable publishing path

## Standard Outputs

- article
- short-form video concepts
- captions
- Pinterest titles and descriptions
- supporting CTA copy

## Publishing Logic

Build the core article first. Then extract and adapt content for each platform without losing message consistency.
