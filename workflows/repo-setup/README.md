# Repo Setup Workflow

This workflow defines the standard process for creating a clean, structured repository.

## Goal

Create new repositories quickly with a consistent structure, naming system, and Git configuration.

## Core Principle

Every repo should:
- have a clear structure
- include documentation
- be Git-ready from the start
- follow the same naming system

## Output

- structured project folder
- initialized Git repo
- connected GitHub repo
- initial commit completed
